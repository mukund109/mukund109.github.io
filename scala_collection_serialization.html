# Scala Map serialization

Ran into a bug today while trying to run a spark job that used a Scala Map to filter out some values from a Spark RDD.

val myMap = Map("foo" -> 2, "bar" -> 4).mapValues( _ * 2 )
rdd.filter( myMap(_) > 5 ) // throws serialization error

The code worked if I didn't use mapValues

val myMap = Map("foo" -> 4, "bar" -> 8)
rdd.filter( myMap(_) > 5 ) // runs

It turns out that mapValues is evaluated lazily. It returns a wrapper around the original Map.

// scala Map implementation

def mapValues[W](f: V => W): Map[K, W] = new MappedValues(f)

protected class MappedValues[W](f: V => W) extends AbstractMap[K, W] with DefaultMap[K, W] {
  ...
  def get(key: K) = self.get(key).map(f)
}

The class MappedValues isn't serializable, which caused the bug.

Moreoever, f is called everytime a value is requested, which might be a problem if f is expensive.

One workaround for this is to force its evaluation:

val myMap = Map("foo" -> 2, "bar" -> 4).mapValues( _ * 2 )
val forcedMap = myMap.map(_)

There does seem to be some progress in finding a fix for this in Scala 2.13.x .
